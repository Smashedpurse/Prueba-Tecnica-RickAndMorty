Planificación de la Prueba Técnica – Rick and Morty
Tecnologías utilizadas

Según los requerimientos del proyecto (framework libre):

React + Vite

Context API + useReducer

Axios

Tailwind CSS

Framer Motion

Objetivo del proyecto

El propósito principal de esta prueba técnica es implementar un sistema de consulta a la API de Rick and Morty utilizando un estado global administrado con Context API y useReducer.
Este enfoque permite:

Realizar búsquedas en tiempo real conforme el usuario escribe.

Manejar reglas de negocio centralizadas, como la gestión y persistencia del estado de elementos favoritos.

Mantener una arquitectura clara, escalable y fácil de mantener.

Buenas prácticas incorporadas:

Componentes modulares y reutilizables.

Manejo adecuado de errores en peticiones HTTP.

Separación clara entre lógica de negocio y UI.

Animaciones no intrusivas mediante Framer Motion.

Diseño responsivo usando utilidades de Tailwind CSS.
